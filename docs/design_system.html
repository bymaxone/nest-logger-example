Example Apps — Design System

The single, project-agnostic visual language shared by every Bymax example application — nest-auth-example, nest-logger-example, and any future nest-*-example. It is forced-dark, built on an orange/blue glass-morphism aesthetic with Geist Sans for body text and a monospace stack for headings, brand, and data. Copy these tokens and component recipes verbatim — the goal is that any two example apps look like one product. This page renders the system live (with hand-written CSS so it works offline) and ends with a step-by-step recreation guide for AI agents.

00 — How to use this guide

One language, every example app

This is the source of the look. The production implementation lives in each app's apps/web as four copied-verbatim files. Never restyle from scratch.

Stack: Next.js 16 (App Router) · React 19 · Tailwind CSS v4 · shadcn/ui new-york · geist fonts · lucide-react icons · sonner toasts. No next-themes — dark is forced by putting dark on <html>.

01 — Color

Brand, semantics, glass & text ramp

Primary is orange #ff6224 (hsl(20.5 90.2% 57.8%)) — it is --primary and --ring. Surfaces are translucent white over near-black; text is white at descending opacity. Use color only with an icon and a label (never color alone) for accessibility.

04 — Tokens (copy-paste)

The globals.css token block

The .dark set is always active (forced dark). Copy verbatim.

Tailwind v4 note. The blocks below show the token values. In a real v4 app, map them to utilities with a @theme inline { … } block (v4 does not auto-load a JS tailwind.config.ts; bridge a kept one via @config './tailwind.config.ts';), keep :root/.dark at top level (not inside @layer base), declare the dark variant with @custom-variant dark (&:is(.dark *)), and use only @tailwindcss/postcss (no autoprefixer). See the full v4-correct block in DASHBOARD.md §15. 
/* app/globals.css */

08 — Severity / log levels

Color + icon + text — never color alone

Log levels (and alert severities) are encoded with a colored left-border, a dot/icon, and the text label together — colorblind-safe. Reuse the same mapping for log rows, the level donut, toasts, and incident badges.

09 — Patterns

The rules behind the look

  • Glass-morphism recipe: surface bg rgba(255,255,255,0.06) + 1px rgba(255,255,255,0.10) border + backdrop-blur-md + rounded-2xl. Primary buttons add the brand glow 0 0 24px rgba(255,98,36,0.4) on hover.
  • Forced dark: no theme toggle. dark on <html>; the .dark token set is the only live one.
  • Mono for data, sans for prose: headings, brand, card titles, metrics, log keys, and table cells are monospace; everything else is Geist Sans.
  • Overview → drill-down: most-important top-left; KPI strip → charts → table; every chart click narrows the table/explorer below.
  • Skeletons, not spinners for content fetches; spinners only for short blocking actions (submit/save).
  • Action-oriented empty states with a primary action — never a blank pane.
  • Accessible severity: color + icon + text together; verify contrast.
10 — AI agent recreation guide

Recreate this look in any example app

Follow these steps to give a fresh apps/web the identical design system.

  1. Install: next@^16 react@^19 tailwindcss@^4 @tailwindcss/postcss geist lucide-react sonner class-variance-authority clsx tailwind-merge. Do not add next-themes or autoprefixer/postcss-import (Tailwind v4 forces dark via a class and auto-prefixes).
  2. Copy app/globals.css (the §4 token block), components.json (shadcn new-york, lucide), and postcss.config.mjs (just @tailwindcss/postcss). In Tailwind v4, map tokens to utilities with a @theme inline block; v4 does not auto-load a JS tailwind.config.ts — if you keep one (e.g. for keyframes), bridge it with @config './tailwind.config.ts'; at the top of globals.css. Put :root/.dark at top level (not in @layer base) and declare the dark variant via @custom-variant dark (&:is(.dark *)).
  3. In app/layout.tsx load GeistSans + GeistMono and put `${GeistSans.variable} ${GeistMono.variable} dark` on <html>.
  4. Add lib/utils.ts with cn = (…i) => twMerge(clsx(i)); scaffold shadcn components (button, card, badge, input, table, tabs, dialog, dropdown-menu, tooltip, sonner, avatar, skeleton, popover, select, command); override button to the pill + brand-gradient variants and card to the glass variant.
  5. Build the shell from §5 (Topbar 64px + Sidebar 250px + main). Reuse the active/idle nav classes exactly; change only the wordmark and the nav items for this app.
  6. Add lib/severity.ts (§8) and reuse it for every status/level surface.
  7. Compose pages overview → drill-down (§6); use skeletons + action-oriented empty states (§9).
Acceptance: place a screenshot of the new app beside any existing example app — the chrome (topbar, sidebar, cards, buttons, fonts, orange brand, glass) must be indistinguishable. Only the page content differs.
